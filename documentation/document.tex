\begin{abstract}
The proliferation of Internet of Things (IoT) devices has introduced significant security vulnerabilities, particularly in resource-constrained environments. Traditional cryptographic primitives often impose prohibitive computational costs. This work implements a secure communication pipeline using the ASCON-128 AEAD algorithm on ESP32-based nodes. We demonstrate a complete lifecycle of sensor data: from binary packetization and authenticated encryption to secure UDP transmission. Our results indicate that ASCON-128 provides a viable balance between robust security (confidentiality and integrity) and the stringent power/performance requirements of embedded hardware.

\keywords{IoT Security \and Lightweight Cryptography \and ASCON-128 \and AEAD \and ESP32.}
\end{abstract}

\section{Introduction}
The Internet of Things (IoT) ecosystem relies on the seamless exchange of data across distributed networks. However, the hardware powering these nodes often lacks the resources to execute standard TLS/SSL stacks efficiently. As a result, lightweight cryptography (LWC) has become a critical area of research.

ASCON-128, the winner of the NIST Lightweight Cryptography competition, offers Authenticated Encryption with Associated Data (AEAD). This paper details the implementation of ASCON-128 to secure environmental telemetry between two ESP32 microcontrollers.

\section{System Architecture}
The architecture follows a Producer-Consumer model optimized for low-latency transmission:
\begin{itemize}
    \item \textbf{Node A (Producer):} Responsible for I2C/SPI data acquisition and ASCON-128 encryption.
    \item \textbf{Node B (Sink):} Responsible for tag verification, decryption, and data logging.
\end{itemize}

\section{ASCON-128 Implementation}
The implementation utilizes the ASCON-128 variant to provide 128-bit security. The AEAD process ensures that Node B can detect if a packet has been tampered with by an adversary. If the 128-bit authentication tag does not match the recalculated value at the receiver, the packet is discarded, effectively mitigating injection and bit-flipping attacks.

\section{Conclusion}
This implementation proves that ASCON-128 is highly suitable for the ESP32 platform, providing enterprise-grade security with negligible impact on transmission latency. Future work will assess power consumption profiles during high-frequency sampling.
